# ROSClaw

**Empirical Safety Analysis of LLM-Controlled Physical AI**

- **Venue:** IROS 2026 (submission accepted)
- **arXiv:** [2603.26997](https://arxiv.org/abs/2603.26997)
- **Replication package:** [github.com/sint-ai/rosclaw-replication](https://github.com/sint-ai/rosclaw-replication)

## Abstract

We present ROSClaw, a benchmark and empirical study of how frontier LLM backends behave when used as the deciding intelligence for physical-AI systems. Across 18,400 trials over four leading frontier backends in a simulated robot fleet (using Gazebo and NVIDIA Isaac Sim) under identical safety envelopes, we measure the rate at which each backend proposes out-of-policy actions.

We find:

- Up to **4.8× spread** in out-of-policy proposal rates across backends
- A persistent **3.4× divergence** between the top two backends that does not converge with scale, with prompt engineering, or with explicit safety instructions
- Specific failure modes (geofence violation, force-cap violation, payload-cap violation) cluster differently across backends

The findings argue against treating frontier LLM backends as fungible substitutes in physical-AI deployments and motivate the protocol-level safety enforcement implemented in [SINT Protocol](/protocol/overview).

## Methodology

Each trial:

1. Spin up a simulated robot fleet (Gazebo or Isaac Sim, depending on the trial class)
2. Issue a scripted scenario via the Protocol's [Bridge](/protocol/layers/bridge) with a capability token encoding velocity, force, payload, and geofence constraints
3. Route through one of the four LLM backends as the agent
4. Capture the proposed action sequence
5. Measure rate of proposals that would violate the token's constraints (caught by the Gate; the simulator never executes them)

18,400 trials cover scenario diversity (workspace, payload, environment, complexity), backend variation, and prompt variation.

## Why this matters for the Protocol

If LLM backends were interchangeable on safety, the protocol-level enforcement we built would be overkill — you'd just pick the safest backend. The data shows backends are not interchangeable, the differences are persistent, and they cluster on specific failure modes.

This is the empirical case for treating the agent as untrusted regardless of provider. The Protocol's [Gate](/protocol/layers/gate) and [capability tokens](/protocol/reference/capability-tokens) are the response.

## Reproducing

```bash
git clone https://github.com/sint-ai/rosclaw-replication
cd rosclaw-replication
pnpm install
pnpm run setup-sim     # downloads Gazebo + Isaac assets (~8 GB)
pnpm run replicate --backend <id> --trials 1000
```

Full reproduction (all 18,400 trials, all four backends) takes approximately 72 hours on a single 8-GPU node. Per-backend partial reproduction is much faster.
