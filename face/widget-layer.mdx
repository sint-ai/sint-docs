# AI Widget Layer

The Widget Layer is the dynamic UI plane around Face. The agent emits `widget.spawn` calls; the Widget Layer instantiates the requested component with scoped props and a TTL.

The principle: chat is not enough. When the agent needs the user to pick from options, confirm a destructive action, edit a draft, or approve a payment — a chat message asking them to "type 1, 2, or 3" is a worse experience than spawning the right UI for the moment.

## How it works

```
Agent output:
  text: "Here are the three contracts that came in this morning."
  widgets: [
    { spawn: "DataTable", props: { rows: [...] }, ttl_seconds: 120 }
  ]
```

Face renders the text via TTS (the avatar speaks it), and the Widget Layer renders the `DataTable` component to the right of the avatar. The component is scoped to its props and times out after the declared TTL.

## Built-in components

| Component | Purpose |
|---|---|
| `ConfirmTile` | Yes/no confirmation with a description |
| `ChoiceList` | Pick one of N options |
| `MultiChoice` | Pick many of N options |
| `DataTable` | Tabular data view |
| `FileDrop` | Accept a file upload |
| `InlineEditor` | Edit a draft (text, JSON, code) inline |
| `ApprovalTray` | Render a pending [Approval](/platform/console/approvals) with approve/deny/modify |
| `ImagePreview` | Display an image |
| `Map` | Display a map with pins |
| `TimePicker` | Pick a time slot |
| `Spinner` | Show progress for a long-running action |

All ledgered: every spawn, every interaction, every dismissal hits the [Evidence Ledger](/protocol/reference/evidence-ledger).

## Custom components

Register custom Widget Layer components for domain-specific UI:

```typescript
import { registerWidget } from "@sint/face/widgets";

registerWidget("PolicyDiff", {
  schema: {
    type: "object",
    properties: {
      before: { type: "string" },
      after: { type: "string" },
    },
    required: ["before", "after"],
  },
  render: (props) => <PolicyDiffView before={props.before} after={props.after} />,
});
```

Once registered, the agent can spawn `PolicyDiff` like any built-in.

## Tier impact

Widget interactions inherit the tier of the action they support:

- A `ConfirmTile` for a T2 action requires the user to confirm before the action proceeds
- An `ApprovalTray` is itself a T2+ surface; the underlying approval determines the tier
- Most other widgets are T0 (informational) by default

## Security

- Widget props are sanitized; arbitrary HTML is rejected
- Custom widgets execute in the host app's React tree; they can do whatever React can do (the security boundary is the host app, not the Widget Layer)
- Widget spawn requests from the agent are validated against the registered schema; mismatches are rejected and ledgered
