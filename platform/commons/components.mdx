# Components

A Component is a reusable [Pipeline](/platform/pipelines/overview) sub-graph. Components encapsulate stages that recur across Operators: PII redaction, OCR, approval routing, retry logic, output formatting.

## Why components

Pipelines compose. A Component is a Pipeline you can drop into another Pipeline as a single node. The host Pipeline doesn't need to know the Component's internal stages; it only needs to know the inputs, outputs, and capability requirements.

## Anatomy

A Component is a directory with:

```
@commons/pii-redactor/
  manifest.yaml          declares inputs, outputs, capabilities, tier impact
  graph.yaml             the Pipeline sub-graph
  policies.yaml          (optional) policies the Component requires
  tests/                 conformance tests
  README.md
```

Example manifest:

```yaml
name: pii-redactor
version: 2.0.1
description: |
  Redacts PII (names, emails, SSNs, addresses, phone numbers) from input
  text before any downstream LLM prompt.

inputs:
  - { name: text,   type: string }
  - { name: locale, type: string, default: en-US }
outputs:
  - { name: redacted_text, type: string }
  - { name: redactions,    type: array<Redaction> }
capabilities:
  - "pii.detect.read"
tier_impact: max(input_tier, T1)   # presence of PII may bump tier
```

## Standard library

Components shipped in the public `@commons/*` namespace:

| Component | Purpose |
|---|---|
| `@commons/pii-redactor` | Strip PII before LLM prompts |
| `@commons/ocr-stage` | Run OCR on scanned input |
| `@commons/classifier` | Generic classifier with confidence floor |
| `@commons/approval-router` | Multi-transport approval routing |
| `@commons/retry-with-backoff` | Retry policy as a sub-graph |
| `@commons/output-formatter` | Schema-aware output rendering |
| `@commons/jurisdictional-flag` | Flag jurisdiction-bound actions |
| `@commons/confidence-gate` | Block low-confidence outputs from autonomous routing |
| `@commons/diff-summary` | Summarize a CRM/document diff for approver review |
| `@commons/canon-fetch` | Deterministic Canon retrieval as a Component |

## Composition

Use a Component in a Pipeline:

```yaml
# pipelines/document-intelligence.yaml
stages:
  - id: ocr
    component: "@commons/ocr-stage@^1.3.0"
    inputs: { source: ${intake.payload} }

  - id: redact
    component: "@commons/pii-redactor@^2.0.0"
    inputs: { text: ${ocr.text}, locale: en-US }

  - id: classify
    component: "@commons/classifier@^1.0.0"
    inputs:
      text: ${redact.redacted_text}
      schema: "doc-types-v3"
      confidence_floor: 0.82
```

Components compose; a Component can use other Components.
