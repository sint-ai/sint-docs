# SINT Console

The Console is where humans observe and intervene in Operator runs. It's the SINT Labs reference [Avatar](/protocol/layers/avatar) surface for operations work — dense, keyboard-navigable, ledger-aware.

## What it's for

- Watch what Operators are doing in real time
- Approve / deny T2+ actions
- Inspect the [Evidence Ledger](/protocol/reference/evidence-ledger) for any tenant you have access to
- Trigger manual runs of a Pipeline
- Pause, resume, or E-stop Operators
- Pull metrics and reports

## Surfaces

| Surface | Purpose |
|---|---|
| **Runs** | Live + historical Operator executions, expandable to per-action ledger rows |
| **Approvals** | The [Approval Tray](/platform/console/approvals) — pending T2+ items routed here |
| **Dashboards** | Per-Operator metrics, latency, throughput, error rate, approval cadence |
| **Ledger** | Direct ledger search and filter (Console is a thin client over the [HTTP API](/protocol/reference/api)) |
| **Pipelines** | Browse, edit, deploy YAML pipelines |
| **Canon** | Browse the per-tenant Canon repo |
| **Memory** | Inspect versioned per-tenant state |
| **Cron** | View and edit scheduled triggers |
| **Tokens** | Issue, attenuate, revoke capability tokens |
| **Settings** | Tenant config, integrations, policy bundles |

## Roles

| Role | Capabilities |
|---|---|
| **Owner** | Full read+write on tenant; manages users, billing, integrations |
| **Operator** | Approve T2+ actions; trigger manual runs; pause/resume; inspect ledger |
| **Approver** | Approve T2+ actions routed to them; read-only on ledger items in their scope |
| **Auditor** | Read-only on ledger and reports; no run control |
| **Engineer** | Edit Pipelines and Canon; deploy via PR + merge |

All access is RBAC-governed and itself ledgered.

## Keyboard model

The Console is built for operators who live in it. Every primary action has a keyboard shortcut.

- `g r` — Go to Runs
- `g a` — Go to Approvals
- `g l` — Go to Ledger
- `/` — Search
- `j / k` — Move down / up in lists
- `Enter` — Open
- `a` — Approve highlighted item (in Approvals)
- `d` — Deny highlighted item (in Approvals)
- `?` — Show all shortcuts

## Ledger view

The Ledger view is a dense table with columns: `id`, `ts`, `kind`, `tier`, `agent_id`, `action_class`, `decision`. Click any row to expand the full payload, the linked Decision/Outcome chain, the trace, and the verifying hash. ProofReceipts (when present) link to the attestation backend.

Filters: tenant, kind, tier, action class, agent, time range, trace ID, free text.

## Run inspection

Each Run is a tree of ledger entries grouped by `trace_id`. The Console renders the run as a graph (intake → plan → gate → execute → ledger → report) with each node clickable to its underlying ledger entry. Failed nodes show inline.

Live runs auto-update via SSE.

## Manual intervention

From any in-flight Run, an Operator with capability can:

- **Pause** — drain in-flight, refuse new
- **Resume** — restart from the last committed ledger entry
- **Modify** the next pending action (subject to attenuation)
- **E-stop** — halt the Run unconditionally; ledger as `estop_fired`

Every intervention is itself ledgered.

## Where it runs

For Agency clients: hosted by SINT Labs at `console.sint.gg`. Per-tenant subdomain (`<tenant>.console.sint.gg`) once tenancy lands. SSO via OIDC; we recommend Okta, Google Workspace, or Microsoft Entra.

For client-hosted: ships as part of the SOP deployment bundle.
