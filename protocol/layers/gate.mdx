# Gate (Layer 2)

The Gate is the policy enforcement plane. Every action sent to the Engine passes through Gate first. Gate's only output is a `Decision`: **allow**, **deny**, or **escalate**.

## Scope

Gate is responsible for:

- Verifying the capability token attached to the action (signature, expiry, revocation status)
- Enforcing [I-T1 attenuation](/protocol/invariants/i-t1-attenuation): the action's required scope must be a subset of the token's granted scope
- Evaluating client-defined policies against the action
- Classifying the action into [tier T0–T4](/protocol/reference/approval-tiers)
- For tier ≥ 2: enqueueing for human approval and returning `escalate`
- Pre-writing the decision to the Engine's ledger queue **before any side effect can fire** ([I-G3](/protocol/invariants/i-g3-ledger))

Gate is **not** responsible for:

- Executing the action ([Engine](/protocol/layers/engine))
- Translating intent ([Bridge](/protocol/layers/bridge))
- Issuing tokens ([Bridge](/protocol/layers/bridge))

## The Decision type

```typescript
type Decision =
  | { kind: "allow"; ledger_id: string; tier: 0 | 1; constraints: AppliedConstraints }
  | { kind: "deny"; ledger_id: string; reason: DenyReason }
  | { kind: "escalate"; ledger_id: string; tier: 2 | 3 | 4; approval_id: string; route: ApprovalRoute };
```

Decisions are immutable once written. The `ledger_id` lets the caller look up the decision in the Evidence Ledger and confirm it was recorded before any execution.

## The decision pipeline

```
Action + Token
    │
    ▼
1. Token inspection (signature, expiry, revocation)         — fail-closed
    │
    ▼
2. Attenuation check (I-T1: required scope ⊆ token scope)   — fail-closed
    │
    ▼
3. Physical constraints check (velocity/force/geofence)     — fail-closed
    │
    ▼
4. Rate limit check                                          — fail-open on infra
    │
    ▼
5. Spend cap check (if applicable)                           — fail-closed
    │
    ▼
6. Policy evaluation (client-defined policies)               — fail-closed on policy
    │                                                          fail-open on infra (I-F1)
    ▼
7. Tier classification (T0–T4)                               — deterministic from action_class
    │                                                          and policy outputs
    ▼
8. Tier ≥ 2 → enqueue for approval, return "escalate"
   Tier 0–1 → return "allow"
    │
    ▼
9. Pre-write Decision to Engine ledger queue (I-G3)          — synchronous; failure → "deny"
    │
    ▼
Decision returned to caller
```

## Policy language

Policies are declarative. The reference DSL ([`@sint/policy-dsl`](https://github.com/sint-ai/sint-protocol/tree/main/packages/policy-dsl)):

```yaml
# policies/default.yaml
- match:
    action_class: "doc.classify"
  effect: allow
  tier: 1
  constraints:
    confidence_floor: 0.82

- match:
    action_class: "doc.summary.write"
    params:
      jurisdiction: { in: ["NY", "CA"] }
  effect: escalate
  tier: 2
  route: { kind: "approval_tray", approver: "duty-attorney" }

- match:
    action_class: "payment.send"
  effect: deny_unless_explicit
  tier: 4
  detail: "Payments require explicit per-action token; root token cannot authorize."
```

Policies compile to a deterministic decision tree at server start. Adding/removing a policy requires a Gate restart (or hot-reload via `pnpm --filter @sint/gateway-server reload-policies`) — the policy table is never mutated under live load.

## Tier classification

| Tier | Definition | Default route |
|---|---|---|
| **T0** | Read-only, no state change | Auto-allow |
| **T1** | Scheduled write within pre-approved scope | Auto-allow |
| **T2** | Ad-hoc write, or write under client-defined "review" condition | Approval tray |
| **T3** | Multi-entity / cross-tenant / multi-recipient | Approval tray + double-confirmation |
| **T4** | Irreversible or regulated (financial settlement, physical motion above threshold, legal filings) | Always human, never autonomous |

## Approval routing

When a decision escalates, the Gate emits an `Approval` event:

```typescript
type Approval = {
  approval_id: string;
  decision_ledger_id: string;
  action: Action;
  route: ApprovalRoute;
  proposed_by: string;          // agent_id
  justification: string;        // agent's rationale, captured by Bridge
  preview: ActionPreview;       // a render of what would happen
  expires_at: number;
};

type ApprovalRoute =
  | { kind: "approval_tray"; approver: string }
  | { kind: "slack"; channel: string }
  | { kind: "email"; recipients: string[] }
  | { kind: "sms"; phone: string }
  | { kind: "webhook"; url: string };
```

Approvers respond via the [Avatar surface](/protocol/layers/avatar), the [Console](/platform/console/approvals), or any custom integration. Approvals carry their own ledger entry that links back to the original decision.

Default approval timeout is tier-defined (T2: 4h, T3: 1h, T4: 24h, all configurable). On timeout, the action is denied; ledgered as `escalation_timeout`.

## Failure modes

| Condition | Behavior | Why |
|---|---|---|
| Token signature invalid | `deny` | Fail-closed on safety |
| Token expired | `deny` | Fail-closed on safety |
| Token revoked | `deny` | Fail-closed on safety |
| Attenuation violation (action scope > token scope) | `deny` | I-T1 |
| Policy infra unreachable (e.g. policy bundle source) | `deny` (with cached policies if available, else `deny`) | Fail-closed on safety |
| Rate limiter unreachable | `allow` if other checks pass | I-F1 — non-safety infra |
| Engine ledger queue unreachable | `deny` | Cannot enforce I-G3 without the ledger |
| E-stop active for tenant | `deny` (all non-terminal actions) | I-G2 |

## What Gate does not do

- **Cache decisions.** Each request is evaluated fresh. Caching would defeat real-time revocation.
- **Modify the action.** Gate either approves the action as-submitted or rejects it. It never "fixes" parameters.
- **Bypass for "trusted agents."** No agent is trusted. The Protocol's whole job is making this assumption explicit.
- **Silent escalation.** Every escalation produces an `Approval` event; no decision is silently held.
