# Document Intelligence Operator

**One-liner:** Ingests documents, classifies them, extracts the fields that matter, and hands a review-ready summary to the right person — under an auditable envelope.

## What it replaces

- Paralegal or intake analyst time on document classification and extraction
- Manual triage queues in contract management, claims, underwriting, compliance intake
- Bespoke Python/regex pipelines that drift on every format change

## Safety envelope

| Tier | Action classes |
|---|---|
| T0 | `ocr.run`, `extract.read` |
| T1 | `classify.doc` (confidence ≥ floor), `extract.fields`, `summary.write`, `route.attorney-queue` |
| T2 | `classify.doc` (confidence < floor), `summary.write` (jurisdictional flag), any out-of-template route |
| T3 | Any write outside the intake namespace |
| T4 | Any document routed to external counsel or regulator |

Hard limits:

- PII redaction before any LLM prompt
- Confidence floor (default 0.82) for autonomous routing
- Per-document-type approver map
- No write-back to client-facing systems without T3 approval
