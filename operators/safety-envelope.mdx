# Safety envelope

Every Operator runs inside the Protocol's safety envelope. This page explains the explicit mapping between Operator stages and Protocol layers, and the rules that don't bend regardless of workflow.

## Layer mapping

| Operator stage | Protocol layer | What happens |
|---|---|---|
| Intake | Bridge (L1) | Inbound work is translated into structured `WorkItem`/`Action` objects; capability token is loaded |
| Plan | (none) | Plan is the agent's proposal; proposals are not actions. Plan is ledgered. |
| Gate | Gate (L2) | Every proposed Action evaluated independently; allow/deny/escalate |
| Execute | Engine (L3) | Decision committed to ledger before execution; Bridge adapter dispatches; outcome ledgered |
| Ledger | Engine ledger | Every event lands here, append-only, hash-chained |
| Report | Avatar (L5) — Console, Face, or any conforming surface | Surfaces outcomes; surfaces approvals to humans |

Layer 4 (Economy) is optional and used only for Operators that meter or settle per action. Most Operators skip it.

## Rules that don't bend

These hold for every Operator, regardless of tenant overrides.

1. **Every action is ledgered.** No "internal" actions skip the ledger.
2. **Every T2+ action waits for human approval.** No "trusted agent" exception.
3. **Token scope is enforced.** No "wildcard for ops" tokens.
4. **PII redaction precedes any LLM prompt.** Configurable per-tenant which classes count as PII; the redaction itself is enforced.
5. **No write to external systems without a capability token scoped to that exact action.** Drift in the system map is a deploy blocker, not a runtime warning.
6. **E-stop universality holds.** Every Operator has an explicit E-stop that bypasses all tier gates.
7. **Confidence floors are real.** Operators that classify or score have a confidence floor. Below the floor, the action is escalated, not "best-effort."

## Per-Operator deviations

Tenant policies can extend (narrow) the default tiers, but cannot widen. A tenant can require T3 approval for an action class the default classifies as T1. A tenant cannot reclassify a T4 default down to T1.

This is enforced at policy-load time. Bundles that attempt invalid widening are rejected.
