# Operators

An Operator is a complete autonomous workflow system. Not a chatbot. Not a copilot. A system that ingests the inputs you already have, executes the steps your team would execute, and writes to the systems of record you already use — under the [Protocol](/protocol/overview)'s safety envelope.

## What "Operator" means here

| It is | It is not |
|---|---|
| A complete workflow, end-to-end | A chatbot bolted to a single tool |
| Backed by a measurable outcome (hours reclaimed, throughput, SLA) | Sold by seat or message |
| Run under the Protocol — every action ledgered, every T2+ approval-gated | Run under "trust the LLM" |
| Built for one workflow at a time | Generic agent framework |
| Owned by SINT Labs (Agency) or by you (Platform) | Loosely-coupled scripts |

## How an Operator is built

1. **Intake** — webhooks, polling, voice, or scheduled triggers. Ingestion is policy-aware: the [Bridge](/protocol/layers/bridge) translates inbound work into `Action` objects.
2. **Plan** — the agent proposes an action sequence. Plans are graphs, not chains; branches and parallel paths are first-class.
3. **Gate** — every action passes through the [Policy Gateway](/protocol/layers/gate). Decisions: allow / deny / escalate, classified by [tier](/protocol/reference/approval-tiers).
4. **Execute** — the [Engine](/protocol/layers/engine) dispatches against the appropriate Bridge adapter.
5. **Ledger** — every decision and outcome lands in the [Evidence Ledger](/protocol/reference/evidence-ledger) before any side effect fires.
6. **Surface** — outcomes report back through [Console](/platform/console/overview), [Face](/face/overview), Slack, email, or your system of record.

## How an Operator is delivered

Two paths:

- **Agency** (today) — SINT Labs builds, deploys, and runs the Operator for you. Two-week fixed-fee Audit, then Build, then Operate. See [sint.gg/agency](https://sint.gg/agency).
- **Platform** (Q3 beta) — self-serve runtime for organizations that want to build and run their own Operators. See [Platform overview](/platform/overview).

Both run on the same stack. Agency Operators are simply Platform Operators that we operate.

## What we measure

Operators are sold against measurable outcomes, not features. Per-Operator metrics:

- **Revenue Operator** — replies/1k touches, meetings booked, cost per meeting, pipeline hygiene score
- **Document Intelligence** — intake-to-summary time, classification accuracy, hours reclaimed
- **Voice** — call completion, average handle time, schedule accuracy, handoff precision
- **Onboarding** — time-from-signed-to-provisioned, first-week oversight hours, downstream tickets
- **CMO** — publishing cadence, qualified inbound, brand-consistency rubric
- **Inbox** — inbox zero cadence, time-to-first-response, loops closed/week

Every metric is computed from the [Evidence Ledger](/protocol/reference/evidence-ledger), so it's auditable.
