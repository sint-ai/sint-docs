# Inbox Operator

**One-liner:** Reads the inbox. Classifies. Drafts. Routes. Sends what's safe to send. Escalates what isn't. Closes loops so nothing rots.

## What it replaces

- EA or ops coordinator time on inbox triage for shared inboxes (support@, ops@, sales@, partnerships@)
- The "I'll get to it tomorrow" email that doesn't get gotten to
- Rules-engine fatigue (Zapier sprawl, filter chains that no one remembers)

## System

```
Email sources
   ▼
Classifier (category + urgency + sender tier)
   ▼
Context assembler (prior threads, CRM, Canon)
   ▼
Draft
   ▼
Gate (tier-based send policy)
   ▼
Send / Queue for approval / Route to human
   ▼
Follow-up scheduler
   ▼
Ledger
   ▼
Weekly loop-close report
```

## Safety envelope

| Tier | Action classes |
|---|---|
| T0 | `inbox.read`, `classify.category`, `assemble.context` |
| T1 | `draft.reply`, `send.reply` (templated, allowlisted category), `schedule.followup` |
| T2 | `send.reply` (ad-hoc), any forward to external domain |
| T3 | `send.reply` (references contracts, pricing, commitments) |
| T4 | Any legal or regulated communication |

Hard limits:

- Never send on behalf of a named individual without T2 approval per-message
- Allowlist of categories that may reply autonomously (per tenant)
- Rate cap per sender domain
- No forwarding to external domains without approval
