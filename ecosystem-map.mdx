## What each piece is

### Layer 0 — The Protocol

- **[SINT Protocol](/protocol/overview)** — the safety standard. Five layers, five invariants. Apache 2.0.
  - **[Bridge](/protocol/layers/bridge)** — agent ↔ actuator translation
  - **[Gate](/protocol/layers/gate)** — policy enforcement, allow/deny/escalate
  - **[Engine](/protocol/layers/engine)** — execution + Evidence Ledger
  - **[Economy](/protocol/layers/economy)** — optional metering / settlement
  - **[Avatar](/protocol/layers/avatar)** — embodied human-in-the-loop surface

### Layer 1 — The Platform (SOP)

The runtime that ships Operators. Every component runs under Protocol.

- **[Console](/platform/console/overview)** — the operator's seat: live runs, ledger, approvals, dashboards
- **[Commons](/platform/commons/overview)** — shared library: skills, components, policies that any Operator can pull from
- **[Memory](/platform/memory/overview)** — versioned per-tenant working state
- **[Cron](/platform/cron/overview)** — scheduled and event triggers
- **[Canon](/platform/canon/overview)** — agent-readable knowledge repo (deterministic retrieval, content-addressed)
- **[Pipelines](/platform/pipelines/overview)** — YAML pipeline definitions that compile to executable graphs

### Layer 2 — The Face

- **[SINT Face](/face/overview)** — voice-first 3D interface
  - [Architecture](/face/architecture) — audio in → ASR → agent → Conversation Compiler → TTS → blendshapes → Widget Layer
  - [Widget Layer](/face/widget-layer) — contextual UI components spawned around the avatar
  - [Conversation Compiler](/face/conversation-compiler) — fast-LLM rewrite for natural speech
  - [Embed guide](/face/embed) — script tag, iframe, SDK

### Layer 3 — The Operators

Each is a complete autonomous workflow system. Six flagship workflows:

- **[Revenue Operator](/operators/revenue)** — outbound, follow-up, pipeline hygiene, quote-to-cash
- **[Document Intelligence Operator](/operators/documents)** — intake, classification, extraction
- **[Voice Operator](/operators/voice)** — inbound/outbound voice for intake, scheduling, triage
- **[Onboarding Operator](/operators/onboarding)** — client/employee onboarding to Day 1
- **[CMO Operator](/operators/cmo)** — content production + distribution against a positioning brief
- **[Inbox Operator](/operators/inbox)** — triage, draft, route, close loops on shared inboxes

### Open-source supporting work

- **[Canon](https://github.com/sint-ai/canon)** — agent-readable repo format, public reference implementation
- **[gstack](https://github.com/sint-ai/gstack)** — skill pack for Claude-family agent stacks
- **[ROSClaw replication](https://github.com/sint-ai/rosclaw-replication)** — replication package for the IROS 2026 paper

### Research

- **[ROSClaw](/research/rosclaw)** — empirical safety analysis of LLM-controlled physical AI (arXiv:2603.26997, IROS 2026)
- **[MCP Security Analysis](/research/mcp-security)** — architectural vulnerability review of MCP (arXiv:2601.17549)

## Two things this map intentionally leaves out

1. **OpenClaw** — our internal 19-agent orchestration setup. It runs SINT Labs internally; it is not a public product. We mention it on [sint.gg/founder](https://sint.gg/founder) and [sint.gg/trust](https://sint.gg/trust) as proof we dogfood, and nowhere else.
2. **M8 Guardrails** — pre-production safety rails we run against Operators in staging. Internal tooling. Available to Platform clients; not a standalone product.
